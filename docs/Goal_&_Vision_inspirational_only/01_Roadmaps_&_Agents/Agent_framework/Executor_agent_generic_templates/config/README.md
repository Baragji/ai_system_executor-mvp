Executor AI — Configuration Directory
This directory contains the core configuration files that control the Executor AI's behavior for this specific project.

session_plan.yaml: This is the master work plan. It defines the objectives and individual "slices" of work for the AI to execute in an ordered fashion. Modify this file to change the AI's goals.

platform_policy.md: This file defines the specific rules and constraints the AI must follow for this project's platform (e.g., web_service, desktop_app). It ensures the AI's output is secure, compliant, and architecturally sound for your environment.
